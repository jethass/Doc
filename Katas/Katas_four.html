Kata Four — Symfony: Dependency Injection - CompilerPass

Story

Given you have a Printer service which print anything through registered Formatters. Allow other bundles to extends your service by adding custom formatters with a compiler pass ans tags.

  • Create your Printer service
  • Create a class wich implements the CompilerPassInterface.
  • Implement the logic in process method.
  • Register the pass in the AppBundle::build method.
  • Create and Configure a JsonFormatter

Subject

Symfony

Dependency Injection CompilerPass

Steps

  1. Create a interface AppBundle\Printing\FormatterInterface with method format($message)

  2. Create a class AppBundle\Printing\Printer.

  3. Implement a method registerFormatter($format, $formatter).

  4. Implement a method print($format, $message) which delegate the job to a dedicated formatter.

  5. Create a class AppBundle\Printing\JsonFormatter implementing the FormatterInterface.

  6. Create services printing.printer and printing.formatter.json

  7. Add a tag named print.formatter to the service printing.formatter.json with an attribute format with json value.

  8. Create a class AppBundle\DependencyInjection\Compiler\PrintingCompilerPass implementing the interface Symfony\Component\DependencyInjection\Compiler\CompilerPassInterface.

  9. Implement the process() with the following.

  10. Check if the service printing.printer exists.

  11. Retrieves the definition of the service printing.printer.

  12. Retrieves the list of services' ids containing the tag printing.formatter.

  13. For each tagged service, register it in the service printing.printer. By adding a call to the method registerFormatter in it definition. Use the attributes array to retrieves the property format defined in the service.xml

  14. Override the method AppBundle::build to add the PrintingCompilerPass in the containerBuilder through the method ContainerBuilder::addCompilerPass. (don't forget to call parent::build).

Help

  • http://symfony.com/doc/current/components/dependency_injection/compilation.html
  • http://symfony.com/doc/current/cookbook/service_container/compiler_passes.html
  • http://symfony.com/doc/current/components/dependency_injection/tags.html

Solution

printing.formatter.json service definition

XML
<service id="printing.formatter.json" class="AppBundle\Printing\JsonFormatter">
    <tag name="printing.formatter" format="json" />
</service>

or

YAML
services:
    printing.formatter.json:
        class: AppBundle\Printing\JsonFormatter
        tags:
            -
                name: printing.formatter
                format: json

PrintingCompilerPass

PHP
<?php

namespace AppBundle\DependencyInjection\Compiler;

use Symfony\Component\DependencyInjection\Compiler\CompilerPassInterface;
use Symfony\Component\DependencyInjection\ContainerBuilder;
use Symfony\Component\DependencyInjection\Reference;

class PrintingCompilerPass implements CompilerPassInterface
{
    /**
     * {@inheritdoc}
     */
    public function process(ContainerBuilder $container)
    {
        if (!$container->has('printing.printer')) {
            return;
        }

        $printerDefinition = $container->findDefinition(
            'printing.printer'
        );
        $formattersTags = $container->findTaggedServiceIds(
            'printing.formatter'
        );

        foreach ($formattersTags as $id => $tagAttributes) {
            foreach ($tagAttributes as $attributes) {
                $printerDefinition->addMethodCall(
                    'registerFormatter',
                    array($attributes['format'], new Reference($id))
                );
            }
        }
    }
}

AppBundle

PHP
<?php

namespace AppBundle;

use AppBundle\DependencyInjection\Compiler\PrintingCompilerPass;
use Symfony\Component\DependencyInjection\ContainerBuilder;
use Symfony\Component\HttpKernel\Bundle\Bundle;

class AppBundle extends Bundle
{
    public function build(ContainerBuilder $container)
    {
        parent::build($container);

        $container->addCompilerPass(new PrintingCompilerPass());
    }
}
